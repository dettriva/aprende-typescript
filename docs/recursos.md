## :toolbox: Ecosistema de herramientas 2026

El ecosistema TypeScript evoluciona rápidamente. Estas son las herramientas que están definiendo el desarrollo con TypeScript en 2026, complementando las herramientas avanzadas de la sección anterior.

| Herramienta | Categoría | Descripción | Enlace |
|-------------|-----------|-------------|--------|
| Bun | Runtime alternativo | Runtime todo-en-uno: bundler, test runner y gestor de paquetes con soporte nativo de TS | [bun.sh](https://bun.sh/) |
| Effect-TS | TypeScript funcional | Framework para programación funcional con tipos: manejo de errores, concurrencia y composición type-safe | [effect.website](https://effect.website/) |
| Drizzle ORM | SQL type-safe | ORM ligero con queries SQL tipadas, migraciones automáticas y zero overhead en runtime | [orm.drizzle.team](https://orm.drizzle.team/) |
| Hono | Web framework type-safe | Framework web ultraligero con tipado end-to-end, compatible con cualquier runtime (Bun, Deno, Node, edge) | [hono.dev](https://hono.dev/) |
| ArkType | Validación | Validación con sintaxis 1:1 con la de TypeScript, máximo rendimiento en runtime y excelente DX | [arktype.io](https://arktype.io/) |

!!! tip "Bun como alternativa a Node.js"
    Bun ejecuta archivos `.ts` directamente sin necesidad de compilación previa. Para desarrollo local y scripts, prueba `bun run archivo.ts` en lugar de usar `tsx` o `ts-node`. Es significativamente más rápido en la mayoría de escenarios.

!!! example "Drizzle vs Prisma"
    Ambos son ORMs tipados, pero con filosofías distintas. **Prisma** usa su propio DSL para el schema y genera un cliente tipado. **Drizzle** define el schema directamente en TypeScript y genera queries SQL reales que puedes inspeccionar. Elige Prisma si prefieres conveniencia, Drizzle si prefieres control y rendimiento.

### ⌨️ Ejercicios de código típicos

Estos son ejercicios que suelen aparecer en pruebas técnicas. Practicarlos te dará soltura tanto con el sistema de tipos como con los patrones funcionales de TypeScript.

1. **Implementar `MyPartial<T>`** — Recrea el utility type `Partial<T>` usando mapped types. Debes hacer que todas las propiedades de `T` sean opcionales.

    ```typescript
    // Tu objetivo: implementar esto desde cero
    type MyPartial<T> = {
      [K in keyof T]?: T[K];
    };
    ```

2. **Implementar `MyPick<T, K>`** — Recrea el utility type `Pick<T, K>` que selecciona solo las propiedades indicadas por `K`.

    ```typescript
    type MyPick<T, K extends keyof T> = {
      [P in K]: T[P];
    };
    ```

3. **Crear un tipo `Result<T, E>` con funciones `map` y `flatMap`** — Implementa un tipo que represente el resultado exitoso o fallido de una operación, junto con funciones para transformarlo de forma segura.

    ```typescript
    type Result<T, E> = { ok: true; value: T } | { ok: false; error: E };

    function map<T, E, U>(
      result: Result<T, E>,
      fn: (value: T) => U
    ): Result<U, E> {
      return result.ok
        ? { ok: true, value: fn(result.value) }
        : result;
    }

    function flatMap<T, E, U>(
      result: Result<T, E>,
      fn: (value: T) => Result<U, E>
    ): Result<U, E> {
      return result.ok ? fn(result.value) : result;
    }
    ```

4. **Tipar una función `pipe` con 3+ overloads** — Crea una función `pipe` que encadene transformaciones con tipos inferidos en cada paso.

    ```typescript
    function pipe<A, B>(value: A, fn1: (a: A) => B): B;
    function pipe<A, B, C>(value: A, fn1: (a: A) => B, fn2: (b: B) => C): C;
    function pipe<A, B, C, D>(
      value: A,
      fn1: (a: A) => B,
      fn2: (b: B) => C,
      fn3: (c: C) => D
    ): D;
    function pipe(value: unknown, ...fns: Function[]): unknown {
      return fns.reduce((acc, fn) => fn(acc), value);
    }
    ```

5. **Implementar `DeepReadonly<T>`** — Crea un tipo recursivo que haga readonly todas las propiedades anidadas de un objeto.

    ```typescript
    type DeepReadonly<T> = {
      readonly [K in keyof T]: T[K] extends object
        ? T[K] extends Function
          ? T[K]
          : DeepReadonly<T[K]>
        : T[K];
    };
    ```

!!! warning "No copies, comprende"
    En una entrevista no tendrás acceso a estas soluciones. Usa estos ejemplos como referencia para **entender el patrón**, luego cierra esta página e intenta resolverlos por tu cuenta. Si no puedes, vuelve a estudiar el capítulo correspondiente.
